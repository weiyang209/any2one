Features

Built for real research workflows

From unified multi-input ingestion to topic splitting, research, review, export, and skill evolution, OneResearchClaw covers the core capabilities required by a full research workflow.

Input
🧭 Unified Multi-Input Ingestion
Supports diverse materials ranging from meeting recordings to PDFs, spreadsheets, and ZIP archives. The system automatically identifies the input type and routes it to the corresponding grounding skill, reducing manual organization, conversion, and preprocessing overhead.
Meeting
🧠 Multi-Topic Meeting Splitting
For long meetings, complex discussions, or multi-topic materials, the system can automatically split content into multiple topics and create independent research branches for each. Every topic can complete grounding, research, summary, and export separately, making it practical to produce several focused reports from one complex meeting.
Research
🎚️ Controllable Research Depth
Supports simple / medium / complex research modes to control literature search scope, opened materials, evidence organization, and analytical depth. It can generate either a fast briefing or a more comprehensive research report.
Research
🔎 Query Confirmation and Targeted Search
Before formal search begins, the system generates candidate queries from the grounded note and lets users confirm, add, remove, or revise them. This keeps literature search aligned with the actual research objective while reducing irrelevant retrieval and wasted tokens.
Review
🧪 Multi-Model Review Mechanism
After report generation, the system enters a review → rewrite loop. Different models or review perspectives can inspect completeness, evidence consistency, logical coherence, and over-claiming, followed by a bounded number of revisions.
Links
☁️ One-Click Cloud Link Ingestion
Supports ingestion and processing of cloud-hosted materials such as arXiv, YouTube, and Bilibili, allowing online sources to enter the same research pipeline directly.
Output
📦 Multi-Format Export
Supports exporting final results as md / docx / pdf / pptx / audio, covering reading, archiving, presentation, and reuse scenarios, with both Chinese and English outputs.
evolution
🧬 Skill-Evolve
Supports evolving subsets of skills from user feedback. Through feedback collection, patch generation, regression checks, and version upgrades, the system can improve continuously instead of staying fixed as a one-off hand-written workflow.

FAQ

Common questions

Placed after Quick Start to help users quickly confirm supported inputs, research depth, the review loop, and delivery options before getting started.

Click a question to expand
What input forms are supported?

Supports meeting audio / video, documents, spreadsheets, slide decks, ZIP mixed-material packages, and remote links such as arXiv, YouTube, and Bilibili. The system first identifies the input type and then routes it to the corresponding grounding workflow.

How should I choose research depth?

research_mode supports simple / medium / complex. Simple is suitable for low-cost briefings, medium is a balanced default, and complex is better when you need broader literature coverage, more opened sources, and deeper analysis.

Do I need to configure extra APIs?

The base workflow does not require an additional reviewer API. By default it can run through a local / Cursor-side workflow. You only need the optional reviewer_api_config if you want each review round to use an external reviewer.

What does the review mechanism do?

After generation, the report enters a review → rewrite loop. The reviewer checks topic alignment, coverage, evidence specificity, analytical depth, structural coherence, and deliverability; the writer revises according to repair actions until the report passes the quality gate or reaches the bounded round limit.

Can export formats be customized?

Yes. output_formats supports md / docx / pdf / pptx / audio, including comma-separated combinations; output_lang controls the final delivery language, such as Chinese or English.

What problem does Skill-Evolve solve?

It turns recurring user feedback into skill modification proposals that are reviewable, verifiable, and reusable, then applies patches, regression checks, and version upgrades to form derived skill variants better aligned with personal preferences.
