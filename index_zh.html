Research Depth

选择你的Research 深度

让 research 模式匹配任务需求，在覆盖度、成本与报告深度之间取得平衡。

模式 · simple
快速概览
快速了解背景并补充核心相关工作，适合轻量汇报或前期探索。
3–5 papers ≥3 opened 2 recent
模式 · medium
均衡研究
兼顾覆盖度、成本和报告深度，适合作为多数任务的默认研究模式。
6–10 papers ≥6 opened 4 recent
模式 · complex
深度研究
适用于跨领域、复杂问题或需要系统性 literature mapping 的任务，产出更深入的研究报告。
10–15 papers ≥10 opened 6 recent

输入支持

支持的输入格式

音频、视频、文档、表格、演示、压缩包与在线链接均可自动识别，并路由到合适的处理流程。

🎙️
会议音频
→ 结构化会议笔记
.mp3 .wav .m4a
使用 WhisperX 进行语音识别和说话人分离，自动生成带说话人归属的 7 段式结构化笔记。
🎬
会议视频
→ 结构化会议笔记
.mp4 .mov .mkv
使用 ffmpeg 提取音频流，再接入 meeting-audio-grounding 流程，生成完整结构化会议笔记。
📄
文档材料
→ 结构化研究笔记
.pdf .docx .md .txt
Docling 深度提取文本、表格、图像、公式与代码块，生成 8 段式结构化笔记。
📊
表格数据
→ 数据分析笔记
.xlsx .csv
提取字段结构、统计摘要、预览行，并自动生成可视化，识别异常值与数据质量问题。
📑
演示文稿
→ 演示结构笔记
.pptx
提取页面结构、标题、正文、图表、表格与备注，分析叙事脉络和关键信号。
📦
压缩归档
→ 档案级汇总笔记
.zip
解压并识别各子文件类型，并行调用相应 grounding skill，最后生成跨文件汇总笔记。

Quick Start

开始上手

三步，从原始输入到完整 research report。

one-report · prompt example 
# Send this prompt to the Cursor Agent

Please use the existing `.cursor/skills/one-report/`
skill to generate a full report from one input file.

First read:
- `.cursor/skills/one-report/SKILL.md`

Input:
- input_path: docs/showcase/inputs/case5/HER2-case5.zip
- output_formats: pdf
- research_mode: complex

Search settings:
- search_backend: cursor
- require_open_link: true
- download_opened_literature: true

Optional:
- output_lang: zh
- transcription_language: zh
1

Prepare your input file

将文件放入 data/raw_inputs/。支持音频、视频、文档、表格、PPT、ZIP 混合包，也支持直接粘贴链接。

2

Send prompt to Cursor Agent

先阅读 .cursor/skills/one-report/SKILL.md，再按示例填写 input_path、research_mode 和 output_formats。

3

Run the full pipeline

Agent 会自动完成 grounding → research → summary → review → export 的端到端执行。

1. Input Routing

识别输入是 ZIP 混合材料包，并记录内部文件如何分发到不同 grounding skills。

manifest.json routed_items.json

2. Grounding

将 ZIP 内部多种材料统一转成后续 research / summary / review 阶段可用的结构化内容。

grounded.md child_outputs/

3. Query Generation

先基于 grounded note 生成候选 query，再确认后进入正式检索，减少无关搜索与 token 浪费。

queries.json queries_confirmed.json

4. Literature Research

在 complex 模式下打开来源、保存证据、整理 paper notes，并生成更扎实的 lit.md。

search_results.json lit.md

5. Summary

整合 grounded note 与 literature result，形成结构化 summary，而不是简单压缩成几条摘要。

summary.md

6. Review → Rewrite

围绕 coverage、evidence specificity、analytical depth 与 deliverability 做有限轮修订。

review_history.json

7. Final Export

当 review 通过后，最终报告写入 research_report.md，并导出为中文 / 英文 PDF。这样用户既能看最终交付物，也能回溯整条 artifact 链。

research_report.md zh/report.pdf en/report.pdf
